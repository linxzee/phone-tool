适用范围说明：

3Dmigoto不针对任何引擎，而是针对DirectX11(DX11)。DX11属于图形API，和游戏引擎不是一个类别，可以认为所有的游戏引擎都要经过图形接口才能完成渲染。目前市面上绝大部分游戏都支持DX11。

重要提示： 有些游戏码下面偷工减料没画东西，去了效果也差，而且并非所有DX11游戏都可以做到去码。播片游戏、某些静态CG的2D游戏的马是直接画在图像上的，这种情况虽然也有办法，但是效果不好且难度大。

第一步：解压安装

Unity游戏安装：

需要看游戏是32位还是64位的，将对应的文件解压到游戏exe所在的目录

虚幻引擎游戏安装：

虚幻引擎游戏正确的exe目录可能是："前面的目录\游戏名\游戏名\Binaries\Win64"，或者"前面的目录\Engine\Binaries\Win64"。如果安装失败请尝试寻找这个目录。

第二步：启动验证

经过上述步骤，3Dmigoto应该已经成功安装，启动游戏exe文件，游戏上方和下方会显示一行绿字，此为启动成功。

第三步：调试操作

启动exe后会看到游戏上下方有绿字，切换到有带马的游戏场景，按小键盘的按键进行调试：

按键说明： "1" - 隐藏下一个纹理（第一组） "2" - 隐藏上一个纹理（第一组） "4" - 隐藏下一个纹理（第二组） "5" - 隐藏上一个纹理（第二组） "7" - 隐藏下一个纹理（第三组） "8" - 隐藏上一个纹理（第三组） "/" - 隐藏下一个纹理（第四组） "*" - 隐藏上一个纹理（第四组） "+" - 取消隐藏

操作流程：

通过"1"和"2"使得码消失，按"3"复制hash值； 通过"4"和"5"使得码消失，按"6"复制hash值； 通过"7"和"8"使得码消失，按"9"复制hash值； 通过"/"和"*"使得码消失，按"-"复制hash值。

第四步：配置文件修改

在d3dx.ini文件第一行加入如下代码：

; 用小键盘"1"和"2"、"4"和"5"找到hash值的情况，优先使用 [ShaderOverride_Mosaic1] hash=找到的hash值 handling=skip ; 用小键盘"7"和"8"、"/"和"*"找到hash值的情况，基本不考虑 [TextureOverride_Mosaic1] hash=找到的hash值 handling=skip

如果是多个作用下的结果，就复制多次，递增数字。命名原理: [ShaderOverride<Name>]

至此去码已经完成，打开游戏应该默认就是已去码的情况。

其他问题解决

隐藏上下绿字：

如果之后不需要hunting，可以按数字键"0"隐藏，更推荐在ini中使hunting=1（0为隐藏）

键位问题：

d3dx.ini中搜NO_VK_DECIMAL VK_NUMPAD1，修改VK_NUMPAD1到9修改为需要的按键

DX12问题解决方案（任选一种）：

1. 尝试变更为DX11运行（首选）：创建快捷方式，目标引号后加（空格）-dx11 2. 把压缩包解压过去的d3d11.dll更名为d3d12.dll 3. 尝试将D3D12文件夹内的d3d12更名为d3d11 4. 尝试外部注入

外部注入教程：

1. 打开：3Dmigoto-1.3.16压缩包，将x64或者x84文件夹解压到对应游戏引擎目录

2. 打开压缩包中的loader，将x64或者x84文件夹中的3Dmigoto Loader.exe解压到第一步的文件夹中

3. 打开d3dx.ini，搜索：target =

[loader] ; ; ;target = dead or alive 6\DOA6.exe ; ; ;module = d3d11.dll

4. 取消如上target和module前面的【;】号，target = 改为游戏本体的exe文件目录

5. 打开3Dmigoto Loader.exe，然后再打开游戏即可

温馨提示： 使用过程中如遇到问题，请参考GitHub项目的Issue页面或相关社区讨论。
